The framing that makes it work

Archimera is an intelligence product, not a high-frequency bot. The edge for judges is the reasoning — "here's a mispriced market, here's why, here's how much to bet" — not microsecond execution. That plays to ML strengths and sidesteps the parts that would eat four days alive: live custody, real order execution, getting liquidated for real.

The stack

Architecture & language split

The horizontal bands are trust boundaries that double as language boundaries. Money-and-keys code runs natively in Rust with an auditable dependency tree; the read-only edge runs in TypeScript on Cloudflare where the blast radius is tiny.

TypeScript edge — reads only Native Rust — signs & settles External — Polymarket & Circle/Arc
The asymmetry that matters

The Worker only ever reads from Polymarket. Only the Rust service ever signs and settles. No secret ever crosses up into the TypeScript band — the entity-secret RSA encryption and every USDC transfer live natively, where the dependency tree is small enough to cargo audit.

The build

Four days, mapped to the diagram

Each day lights up specific boxes. Build the gray read-path and teal Worker first, then the coral brain, then wire the coral wallet engine to Circle/Arc, then the teal frontend and — equally weighted — real users.

DAY1read path

Data spine + the estimate

touches → Polymarket Gamma · Read Worker (KV) · brain stub
  • Stand up the read Worker. Hit gamma-api.polymarket.com/markets, pull active markets by 24h volume, normalize price → implied probability. Public, no auth — this part is trivial.
  • Build the intellectual core. A function that produces an independent probability for one market: reads the question + recent context, outputs a probability, a confidence interval, and a written rationale.
  • The rationale IS the product. Get one market fully end-to-end: market in → your-prob + market-prob + edge out. No UI yet.
End of day One market, one honest probability estimate with a trace. The hard thinking starts here, not the plumbing.
DAY2the brain

+EV detection & Kelly sizing

touches → Probability brain · Kelly sizing (Rust core)
  • Flag the mispricings. Where |your_prob − market_prob| clears a threshold, rank by edge.
  • Size with fractional Kelly. Use quarter-Kelly, never full — full Kelly is famously too aggressive and a judge who trades will notice raw Kelly instantly.
  • Highest-learning day. Kelly + EV are new conceptual territory — give them real time. Output a ranked list of +EV picks, each with a size and an attached reasoning trace.
End of day A ranked feed of sized, reasoned picks. This is the agentic-sophistication core — 30% of the score.
DAY3make it act

Arc / Circle — make it move money

touches → Wallet engine · Circle + Arc testnet
  • Encrypt the entity secret. RSA-OAEP, SHA-256 for both the hash AND MGF1 — the detail generic helpers get wrong. Fresh ciphertext per request; never cache it.
  • Create a dev-controlled wallet on Arc. POST /v1/w3s/developer/wallets with accountType: "SCA" — matches Circle's known-good Arc path. Fund from the testnet faucet.
  • Allocate + pin. On an accepted pick, generate fresh ciphertext, move the Kelly-sized USDC, and record the reasoning trace as a settlement record on Arc (cents to pin the full trace).
End of day The agent doesn't just recommend — it places capital. This is what converts "smart dashboard" into "agent that interfaces with markets."
DAY4ship + users

Frontend, traction, demo

touches → Pages frontend · Cron scan · the submission
  • Build the feed UI. Each card: market price vs. agent probability, the edge, the Kelly size, an expandable reasoning trace. Wire the Cron Trigger so the feed visibly refreshes itself — that's the "monitors 24/7" story.
  • Get real users — this is 30% of the score. Equal weight to everything you built. Post in the Canteen Discord; get 5–10 people to open the link, fund a testnet wallet, look at picks.
  • Record + submit. 3-min Loom (mispriced market → why → position placed in USDC on Arc), live link, public GitHub repo with a README a judge can follow cold.
End of day Submitted early. You can submit as many times as you like — submit, then keep gathering traction.

An honest reality check

Probability quality is where this lives or dies, and being genuinely calibrated in four days is hard — your estimates may not beat the market. That's okay. Judges weight agentic decision-making and reasoning legibility heavily: a well-reasoned wrong probability with an honest confidence interval demos better than a black-box right one. Lean into showing how it thinks.
